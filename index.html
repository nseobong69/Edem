📖 How It Works & Deployment Guide

Does my file stay on my device?
Yes — for Audio & Video mode, your file is processed entirely inside your browser. Nothing is sent anywhere. Phones are slower for file processing — a laptop or desktop is much better.

For Live Capture mode, your voice is processed by your browser's built-in speech engine. On Chrome, this is sent briefly to Google's servers — but not stored permanently.

Audio & Video vs Live Capture — which should I use?

🎵 Audio & Video

  • Works on any audio/video file
  • Fully offline after engine loads
  • Better accuracy (90–95%)
  • Timestamps included
  • Audio never leaves device

🔴 Live Capture

  • Instant — no engine download
  • Works great on phones
  • Real-time as you speak
  • Good for meetings, notes
  • Needs internet + Chrome/Edge

Is it 100% accurate?
No tool is truly 100%. The Balanced engine gives ~88–93% accuracy on clear speech. Live Capture is about 85–90%. Background noise, fast speech, and accents reduce accuracy. Always review the output.

🚀 Deploy for Free (GitHub Pages)

1
Create a free GitHub account
Go to github.com → Sign Up. It's free. Use any email.
2
Create a new repository
Click the + icon → "New repository". Name it anything (e.g. edem). Set it to Public. Click "Create repository".
3
Upload the HTML file
On your new repo page, click Add file → Upload files. Drag in this HTML file. Rename it to index.html. Click "Commit changes".
4
Turn on GitHub Pages
Go to Settings → Pages (left sidebar). Under "Source", select Deploy from branch. Choose main branch, folder / (root). Click Save.
5
Your app is live!
Wait ~60 seconds. Your URL will be: yourusername.github.io/edem. Share this link with anyone — works on any phone or computer, completely free.
💡 That's it! No coding needed. EDEM is now a real website anyone can use. Update it anytime by uploading a new HTML file.
